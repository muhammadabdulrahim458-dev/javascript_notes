## Why This Order Matters

This roadmap is not a list of random syntax tricks. Each module builds directly on the mental model from the one before it ‚Äî starting with how the **JavaScript Engine actually executes code in memory**, and ending with the metaprogramming patterns used in production-grade systems.

<Callout title="Sequencing Rule" type="info">
  Do not skip ahead. Module 5 (Async Architecture) only makes sense once the Call Stack, Execution Contexts, and Closures from Modules 1‚Äì3 are solid.
</Callout>

## Curriculum

<Steps>

<Step>
### Module 1 ‚Äî The Engine Runtime & Compilation

Understand exactly how the browser/Node.js executes your code before you write a single line.

- **JS Engine Architecture** ‚Äî Parser, AST, Interpreter (Ignition), JIT Compiler (TurboFan)
- **Execution Contexts** ‚Äî Global vs. Function Execution Context
- **Two-Phase Execution Lifecycle** ‚Äî Creation Phase (memory allocation) vs. Code Execution Phase
- **Hoisting Mechanics** ‚Äî function declarations vs. `var` vs. `let`/`const`
- **Temporal Dead Zone (TDZ)** ‚Äî why block-scoped variables fail before initialization
- **The Call Stack** ‚Äî stack frames, LIFO tracking, stack overflows
</Step>

<Step>
### Module 2 ‚Äî Memory Management & Types

- **Primitive vs. Reference Values** ‚Äî stack storage vs. heap pointers
- **Garbage Collection** ‚Äî Mark-and-Sweep, detached DOM nodes, forgotten intervals
- **Deep Type Coercion** ‚Äî implicit vs. explicit casting, `==` vs `===`, `ToPrimitive`
- **Accurate Type Checking** ‚Äî limits of `typeof`/`instanceof`, `Object.prototype.toString.call()`
- **Immutability Patterns** ‚Äî deep vs. shallow cloning, `structuredClone()`, `Object.freeze()` vs. `Object.seal()`
</Step>

<Step>
### Module 3 ‚Äî Scope, Closures & Functional Architecture

- **Lexical Scoping Chains** ‚Äî static scoping vs. runtime evaluation
- **Closures Deep Dive** ‚Äî lexical environments, persisting state, data privacy
- **The `this` Keyword** ‚Äî global, implicit, explicit, and `new` binding
- **Explicit Binding** ‚Äî `.call()`, `.apply()`, `.bind()`
- **Advanced Functional Patterns** ‚Äî Higher-Order Functions, pure functions, currying, partial application, composition
</Step>

<Step>
### Module 4 ‚Äî Data Structures & Advanced ES6+ Syntax

- **Declarative Arrays** ‚Äî custom implementations of `.map()`, `.filter()`, `.reduce()`, `.some()`, `.every()`, `.flatMap()`
- **Destructuring & Rest/Spread** ‚Äî nested patterns, cloning, parameter bundling, defaults
- **Keyed Collections** ‚Äî `Map` vs. plain objects, `Set`, `WeakMap`/`WeakSet`
- **Strings & Regex** ‚Äî template literals, tagged templates, lookaheads, capture groups
- **Modern Operators** ‚Äî optional chaining (`?.`), nullish coalescing (`??`), short-circuiting
</Step>

<Step>
### Module 5 ‚Äî Asynchronous Architecture

Critical for AI event loops and streaming responses.

- **The Event Loop** ‚Äî Call Stack, Web APIs/Node C++ APIs, MacroTask Queue, MicroTask Queue, Render Queue
- **Microtask Prioritization** ‚Äî why Promises interrupt `setTimeout`/`setInterval`
- **Evolution of Async Flow** ‚Äî Callbacks ‚Üí Callback Hell ‚Üí Promises ‚Üí Async/Await
- **Promise Architecture** ‚Äî building a custom Promise from scratch (Pending, Fulfilled, Rejected)
- **Async Orchestration** ‚Äî `Promise.all()`, `Promise.allSettled()`, `Promise.any()`, `Promise.race()`
- **Streams, Iterators & Generators** ‚Äî custom iterators, `function*`/`yield`, `for await...of`
</Step>

<Step>
### Module 6 ‚Äî Object-Oriented JavaScript & Prototypes

- **The Prototype Chain** ‚Äî `__proto__` vs `prototype`, shadowing, lookup path
- **Prototypal Inheritance** ‚Äî `Object.create()`, constructor functions
- **ES6 Class Syntax** ‚Äî classes, instantiation, compilation patterns
- **OOP Pillars** ‚Äî `extends`/`super`, private fields (`#private`), static members, polymorphism
</Step>

<Step>
### Module 7 ‚Äî DOM Engineering & Browser APIs

- **DOM Tree Composition** ‚Äî node hierarchies, live HTMLCollections vs. static NodeLists
- **Optimized Manipulation** ‚Äî `querySelector`, reflow/repaint triggers
- **Event System Workflow** ‚Äî capturing, target, bubbling phases
- **High-Performance Patterns** ‚Äî event delegation, `.stopPropagation()`, `.preventDefault()`
- **Browser Storage** ‚Äî `localStorage`, `sessionStorage`, cookies, `IndexedDB`
- **Modern Fetching** ‚Äî Fetch API, `AbortController`, CORS, stream chunk parsing
</Step>

<Step>
### Module 8 ‚Äî Metaprogramming & Advanced Patterns

- **Proxy & Reflect APIs** ‚Äî intercepting behavior, trapping gets/sets, reactivity wrappers
- **Performance Algorithms** ‚Äî debouncing and throttling middleware
- **Memoization Strategies** ‚Äî global caching for expensive operations
- **Design Patterns** ‚Äî Module, Singleton, Factory, Observer
- **Error Architecture** ‚Äî custom `Error` classes, `try...catch...finally`, unit testing with Jest
</Step>

</Steps>
